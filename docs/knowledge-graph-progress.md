# OmniLit 文献知识图谱重构进度

本文件记录 `knowledge-graph-vision.md` 长期目标的可交付增量。每轮只推进一个可独立验收的纵向切片。

## 当前阶段

阶段一“统一图谱基础”和阶段二“现代化图谱探索器”已有可运行基础：统一节点/关系结构、抽取与规范化、证据和置信度、质量摘要、搜索筛选、布局、详情面板、比较视图与多格式导出均已接入产品入口。

阶段二的现代化图谱探索器、阶段三的领域概览和阶段四的时间演化已形成连续工作流；下一步进入阶段五“聚类和趋势分析”，补齐共被引、文献耦合、密度与桥接节点分析。

## 本轮交付：证据驱动的图谱构建回放

验收目标：用户可以从 PDF 原文证据开始，按阅读顺序观察实体和关系逐步进入图谱；最后回到完整文档浏览状态。

已完成：

- 将节点和关系按页码、区块位置及证据文本组织成稳定的回放事件。
- 关系只有独立证据时仍可生成事件，并同步揭示非论文端点。
- 回放期间控制器只暴露已揭示节点和边，始终保留中心论文节点。
- 原文页、证据框、关系触发词、图节点之间建立同步视觉反馈。
- 支持播放、暂停、单步、重新演示和明确的完成状态。
- 完成后切换为惰性渲染的全文浏览，避免一次渲染全部 PDF 页面。
- 对畸形页码和坐标做容错，避免单条坏证据中断回放。

验证：

- 默认环境完整测试：413 项通过，98 项按可选依赖条件跳过。
- PySide6 环境知识图谱相关测试：26 项通过，1 项因缺少 PyMuPDF 跳过。
- `qmllint` 解析成功，无 QML 语法错误；现有上下文属性产生未限定访问警告。

## 本轮交付：种子论文与按需邻居展开

验收目标：单篇文献图谱不再默认渲染完整缓存图；用户从种子论文出发，按关系类型和分页逐步揭示当前任务需要的上下文。

已完成：

- 控制器分别维护完整缓存图与当前探索子图，搜索、筛选、回放和比较图保持兼容。
- 单篇图谱默认只显示种子论文；比较图继续显示完整对比上下文。
- 支持唯一邻居分页，同一邻居存在多条关系时一次揭示全部匹配边，不重复计数节点。
- 支持引用、被引、作者、机构、主题、期刊和全部关系七种展开模式。
- 提供 `loading`、`ready`、`empty`、`error` 状态及可重复调用的重试入口。
- 双击节点可展开下一页邻居；详情面板显示各关系类型的可展开数量。
- 支持重置探索，恢复到种子论文状态；搜索可以定位尚未手动展开的节点。
- 作者、机构和期刊已升级为一等语义实体，新增 `AUTHOR_OF`、`AFFILIATED_WITH` 和 `PUBLISHED_IN` 关系。
- 构建器版本升级为 7，旧缓存会通过现有指纹机制后台重建。

## 本轮交付：可解释最短路径分析

验收目标：用户可以在当前探索子图内选择两个端点，按方向和关系类型计算最短路径，并理解路径为何成立。

已完成：

- 实现确定性的广度优先最短路径，支持有向和无向遍历。
- 支持按单一关系类型过滤，关系选项只来自当前探索子图。
- 对端点失效、无路径、回放冲突和访问节点上限提供明确状态。
- 路径结果包含节点、边、方向、关系类型、置信度、方向依据和逐步自然语言解释。
- 路径节点和边使用独立高亮，起点和终点分别使用成功色与错误色标识。
- 路径节点优先于普通过滤器和节点上限显示，但不会加载探索子图之外的数据。
- 单篇与多文献对比页面均接入端点选择、方向、关系过滤、计算、清除和步骤面板。
- 路径端点、方向和关系过滤纳入研究视图快照；恢复后会在当前图上重新计算。
- 建立 10,000 节点线性图路径性能测试，限制异常大图搜索访问量。

## 本轮交付：撤销、重做与默认视图恢复

验收目标：用户可以安全回退或重新执行图谱探索中的语义操作，并一键恢复种子论文默认上下文。

已完成：

- 建立容量为 60 的轻量撤销/重做栈，只保存状态和 ID，不复制节点、关系正文或完整图谱。
- 邻居展开、筛选、搜索、节点/关系选择、密度、文献排序、路径端点/方向/过滤和研究视图恢复均进入历史。
- 连续搜索输入在 0.9 秒窗口内合并为一个操作，悬停、动画和派生路径计算不进入历史。

## 本轮交付：高质量图片导出

验收目标：用户可以把当前研究画布或完整探索子图导出为无损 PNG，并且导出过程不永久改变正在使用的视图。

已完成：

- 单篇图谱与多文献对比图共用图片导出对话框和同一事务接口。
- 支持“当前视口”和“完整探索图谱”两种范围；完整范围直接读取探索上下文，不受当前搜索、筛选、总览节点上限影响。
- 支持 1× 至 4× 分辨率、当前主题或透明背景，以及跨平台安全文件名和自动重名避让。
- 导出画面保留节点、关系、标签、路径高亮和主题颜色，同时隐藏设置、缩放、全屏和小地图等交互控件。
- 完整范围导出临时切换到学术布局、显示标签并自动适配画布；完成或失败后恢复原布局、缩放、平移和全部视觉参数。
- 对空图、构建回放冲突、重复任务、超大尺寸、截图启动失败、写入失败、无效 PNG 和回调错配提供明确错误状态。
- 写入后校验 PNG 签名，并生成同名 `.png.json` 清单，记录范围、倍率、背景、图谱指纹和导出时间。
- 单边最大尺寸限制为 16,384 像素，总像素限制为 1 亿，避免高倍率导出耗尽图形内存。

验证：

- 默认环境完整测试：450 项通过，112 项按可选依赖条件跳过。
- PySide6 环境知识图谱相关测试：108 项通过，1 项因缺少可选依赖跳过。
- 新增导出组件及相关页面通过 `qmllint`，无 QML 语法错误；保留既有上下文属性未限定访问警告。

## 本轮交付：大图分层加载与层级细节

验收目标：语义图可以保留数千至一万个当前可见节点，但 QML 只创建与当前缩放、视口和任务相关的有限图元；降级渲染不改变分析结果或完整导出内容。

已完成：

- 新增独立纯 Python 渲染投影层，将“语义可见集合”和“实际 QML 渲染集合”分离。
- 根据缩放自动切换 `overview`、`normal`、`detail` 三个层级，默认节点预算分别为 240、480、900，硬上限为 1,200。
- 学术布局按真实视口、缩放、平移和 120px 预加载边界裁剪；总览、径向和焦点布局避免使用不匹配的空间坐标裁剪。
- 超出预算的节点按语义类别和空间网格聚合，生成稳定聚合节点、成员数量、样本 ID、平均置信度和汇总关系。
- 点击聚合节点会切换到学术布局并放大对应区域，下一次投影自然展开该区域细节。
- 选中节点、选中关系端点和最短路径节点/边具有渲染优先级，不会因层级预算丢失。
- 当前视口图片导出使用屏幕实际投影；完整探索图谱导出继续读取独立语义图和完整布局。
- 文献列表改为读取不受 QML 节点预算影响的语义集合；保存视图通过已保存的缩放、平移和布局自动重建投影。
- 视口变化使用独立 `renderChanged` 信号，不触发文献列表、详情面板、历史栈和其他控制器属性的全量更新。
- 渲染状态公开语义节点数、视口候选数、真实/聚合节点数、裁剪数、边数、层级、耗时和 120ms 预算超限状态。
- 对缺失布局、非法坐标、非数值重要性/置信度和孤立节点提供确定性容错位置与降级结果。

## 本轮交付：可解释主题聚类与领域概览

验收目标：用户可以从当前筛选文献集合查看主要研究主题、子主题、代表论文和增长情况，理解论文为何被归入主题，并从主题气泡直接进入相应局部图谱。

已完成：

- 使用 TF-IDF 加权语义向量和确定性连通分量聚类；相同输入和不同输入顺序产生相同主题、归属、布局和缓存键。

## 本轮交付：文献时间演化与关键引文路径

验收目标：用户可以按年份观察论文和主题如何出现、扩张和连接，沿真实引文追踪关键知识路径，并将任意时间窗口同步到主题气泡和局部图谱。

已完成：

- 新增版本 1 时间演化数据契约，统一输出论文时间属性、年度事件、主题序列、有向引文、关键路径、转折点和诊断信息。
- 时间模型复用主题地图的论文归属和馆藏引文，不把语义相似关系冒充真实引用。
- `CITES` 始终保存“引用论文 → 被引论文”的原始方向；关键路径明确反向解释为“被引论文 → 后续引用论文”的知识影响方向。
- 年份晚于引用论文的被引记录标为时间冲突，不进入关键路径；缺失年份引文和同年循环引文分别提供降级计数。
- 同年无环引文按照真实方向进行局部拓扑排序，不再使用论文 ID 字典序猜测先后；只有检测到循环时才确定性忽略回边。
