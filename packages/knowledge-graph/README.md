# @omnilit/knowledge-graph

桌面与网页共享的知识图谱产品模块。业务状态只依赖 `@omnilit/shared-schema`，渲染器通过
`GraphRenderer` 隔离；当前唯一引擎实现为 `G6GraphRenderer`。

当前纵向切片提供：

- 统一 Reducer：GraphData、筛选、节点/边选择和悬停状态；
- O(nodes + edges) 的关键词与节点类型筛选；
- G6 v5 圆形布局、画布缩放、拖拽、节点/边事件、适应视图和图片导出接口；
- 节点类型图例、画布选择、可搜索键盘列表、节点/边详情及证据文本；
- 可取消的关系模式邻居分页、按 ID 去重合并，以及图谱/文献列表双向选中和画布聚焦；
- Local Agent 概览/标准/细节投影、聚合节点钻取、投影取消与性能诊断；
- G6 画布真实尺寸、缩放和平移反馈；投影层级相对当前相机应用语义缩放；
- 与 QML 共用版本 2 图谱视图：命名保存、列表、恢复、删除，以及筛选、选择、层级和相机恢复；
- 大列表每批 100 个节点，250 节点以上按重要性降低标签密度；
- 加载与服务错误由调用入口负责，模块负责原始空图和筛选空状态；
- 100、1k、5k、10k 节点状态筛选基线测试。
- 共享知识演化时间轴：范围选择、播放/暂停、主题里程碑、关键引用路径、主题速度比较，以及时间轴论文与 G6/文献列表联动；所有演化计算来自 Local Agent 或固定 GraphTimeline，不在 React 主线程重算。

模块不直接访问 API、Qt、数据库、PDF 或图算法。大型布局、聚类、PageRank、路径和抽样
必须由 Web Worker、Local Agent 或 Cloud Graph Engine 预计算后作为协议数据传入。

当前尚未迁移交互式聚类、趋势分析和产品导出流程；
因此不能将本切片视为里程碑 3 完成。
